# 字幕引擎 2.0 质量验收

这份清单是发布前的固定门槛。`pnpm quality` 会执行前端类型检查、生产构建、Rust 单元测试和 Clippy；下面的运行态验收需要在 Windows 桌面环境手动完成。

## 自动化门槛

- `pnpm quality`：前端构建、17+ 个 Rust 测试、`cargo clippy -- -D warnings` 全部通过。
- 字幕源解析测试覆盖：滚动字幕去重、状态提示暂停、状态提示解除、Unicode 截断、日文 PowerShell 输出。
- LLM 请求测试覆盖：OpenAI-compatible endpoint 规范化、DeepSeek V4 thinking 显式关闭、上下文提示合并。

## Windows 运行态门槛

1. 启动/停止字幕 20 次：窗口、托盘、后台进程均能回到正确状态，不残留重复 overlay。
2. 静音 30 秒后恢复声音：状态显示“暂无新声音”，新句子到来后在 1 个轮询周期内恢复；不显示 Windows 性能提示作为译文。
3. 结束 `LiveCaptions.exe` 再重新打开：健康条进入“自动重连”，最多 5 秒退避后恢复；重连次数和最后更新时间可见。
4. 连续产生快于 LLM 的字幕：界面只显示最新片段，不出现持续向下堆积的旧翻译；取消请求后无未处理任务增长。
6. 选择翻译在屏幕四边触发：结果窗完整落在当前显示器工作区；点击其他位置不会关闭，可拖动标题栏或右上角关闭。
7. LLM 请求耗时：实时字幕超时上限不超过 5 秒，划词翻译不受该总时限约束；健康条显示首 token 延迟，日志不包含 API Key。
8. 运行 30 分钟后观察任务管理器：进程数不增长，内存无持续线性增长，日志文件按会话追加且单次日志缓存最多 500 条。
9. 发布版验收：直接启动 `target/release/livecaption.exe` 不出现终端窗口；dev、raw release、安装版均读取 `%LOCALAPPDATA%\com.dimfi.livecaption\settings.json`，重新构建不会改写其中的模型、拖动方式或颜色。

## 发布判定

自动化门槛全部通过，且运行态门槛 1–9 均通过，才标记字幕引擎 2.0 为可发布版本。任一项失败时，保留日志和复现步骤，禁止以“重启软件”作为验收结果。
